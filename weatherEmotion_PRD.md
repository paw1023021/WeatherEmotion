# Weather Emotion PRD (Product Requirements Document)
20201651 박수빈

## 1. 개요

**Weather Emotion**은 날씨 변화와 사용자의 감정 상태를 연결하여, 그날의 기분과 기상 조건에 적합한 “추천 활동”을 제안하고 기록을 도와주는 iOS 앱이다.

### 핵심 가치 제안
- **문제**: 날씨 변화로 기분이 흔들리지만 이를 관리하거나 활용할 수 있는 앱이 부족함  
- **해결책**: 사용자의 감정 + 실시간 날씨 정보를 결합한 맞춤형 활동 추천  
- **차별점**: 감정 기록, 활동 체크, 날씨 기반 분석을 한 번에 제공하는 통합형 감정·활동 기록 앱  

---

## 2. 타겟 사용자

### 주 사용자
- 20~30대 감정 기록 관심층  
- 날씨와 기분 변화를 민감하게 느끼는 사용자  

### 페인포인트
- 감정 기록 앱과 날씨 앱이 분리되어 비효율적  
- 날씨 변화에 따른 기분·행동 가이드를 제공하는 앱이 부족  
- 오늘 무엇을 해야 기분이 좋아질지 고민됨

## 4. 기술 스택

### 개발 환경
- **iOS**: 16.0+
- **아키텍처**: MVVM  
- **데이터**: CoreData  
- **API**: 외부 날씨 API
- **AI**: Google Gemini API   

### 주요 라이브러리
- Swift Charts (시각화)
- SwiftUI Components (UI) 
- JSON Parsing (날씨 API 응답 데이터를 Swift 구조체로 변환)
---

## 5. Gemini API 통합 전략

### 프롬프트 전략
- **시스템 프롬프트**: 날씨와 감정 변화에 공감하고 따뜻하게 조언하는 톤  
- **컨텍스트**: 현재 날씨 정보(상태·온도), 사용자가 선택한 감정, 최근 활동 기록  
- **응답 형식**: 활동 추천·설명·태그를 포함한 JSON 구조(카드 UI 파싱 용이)

---

## 6. UI/UX 원칙

### 디자인 시스템
- **색상**: 날씨 상태에 따라 변화하는 파스텔 기반 컬러 사용 
- **타이포**: SF Pro  
- **인터랙션**: 직관적 버튼/스와이프  
- **접근성**: Dynamic Type, 다크모드 지원  

### 핵심 화면
1. 홈: 날씨·기분 선택  
2. 추천 활동: 카드 제공  
3. 활동 기록: 체크리스트  
4. 통계: 감정 변화 및 활동 그래프  

---

## 7. 사용자 스토리

1. **기분 선택**
   - 사용자는 오늘의 기분을 선택하여 기록한다.

2. **날씨 확인**
   - 현재 위치 기반 날씨 정보를 확인한다.

3. **추천 활동 확인**
   - 날씨·감정 조합에 따라 활동 카드를 확인한다.

4. **활동 수행 기록**
   - 활동 완료 여부를 체크하여 기록한다.

5. **통계 확인**
   - 월간 감정·활동 참여도를 시각적으로 확인한다.

---

- 시각적 통계 제공  

### 호환성
- iOS 16.0 이상 지원 
- 다크모드 지원
